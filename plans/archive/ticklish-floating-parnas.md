# Plan: Generate an sk-ant-oat01- OAuth Token

## Context
The user wants to obtain a Claude Code OAuth Access Token (prefixed `sk-ant-oat01-`). These tokens are generated during the browser-based OAuth login flow with a Claude.ai account (Pro or Max). They are distinct from regular API keys (`sk-ant-api01-`).

## How sk-ant-oat01- tokens work
- Generated by Claude Code when you authenticate via Claude.ai OAuth (not the Anthropic Console)
- Stored locally after the interactive login flow
- Used by Claude Code to make API calls on behalf of your Claude.ai subscription

## Steps to generate the token

### Step 1: Run the interactive login
```bash
claude
```
If already logged in, log out first:
```bash
claude logout
```

### Step 2: Follow the OAuth flow
- Select "Login with Claude.ai account" when prompted
- A browser window will open to claude.ai
- Authorize the Claude Code application
- The CLI will exchange the authorization code for an `sk-ant-oat01-` token

### Step 3: Retrieve the stored token (if needed for external use)
After login, Claude Code stores credentials locally. On Windows:
```
%USERPROFILE%\.claude\.credentials.json
```
The token will be stored there and starts with `sk-ant-oat01-`.

## Alternative: Use a regular API key instead
If you need a key for direct API access (not tied to a Claude.ai subscription), use the Anthropic Console at console.anthropic.com to generate an `sk-ant-api01-` key instead.

## Verification
After login, run `claude` — if it starts without prompting for authentication, the token was successfully generated and stored.
