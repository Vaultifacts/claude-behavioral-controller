# /security-scan — Security Audit

OWASP-style security audit covering secrets, injection vectors, auth gaps, sensitive data exposure, misconfiguration, and dependency vulnerabilities.

## Arguments
- `$ARGUMENTS` — Optional: specific area or file to focus on (e.g., "auth", "api", "src/handlers"). If omitted, scans the full project.

## Instructions

Use the `security-scan` skill to execute this command.
